## 하필 라즈베리파이?

요새는 개인 서버를 사용하는 방법이 수두룩하다. AWS, GCP 등의 Cloud 플랫폼을 통해 다양한 성능의 VM을 골라 사용해도 되고, 직접 컴퓨터를 구매한다면 라즈베리파이보다 고성능인 중국 미니 PC들도 구매할 수 있다. 그럼에도 라즈베리파이가 가지는 명확한 장점이 있고, 해당 장점을 다른 클라우드/미니PC를 사용해 얻는 장점보다 좋다고 판단해, 라즈베리파이를 사용하게 되었다.
![[Pasted image 20260421113223.png]]
개인적으로 느낀 장점들은 아래와 같다

### 1. 비용

가장 큰 이유. 한두달 잠깐 쓰고 마는 것이 아니라 블로그 + 프로젝트 테스트 + a를 돌리려는 목적이라 장기간 사용해도 부담없는 서버를 원했다.
구체적으로 원했던 스팩은 아래와 같다.

- 최소 100GB 이상의 저장공간
- Go의 `net/http`나 FastAPI 등으로 띄운 서버를 적어도 1개 이상 운영 가능한 사양
- 최소 1개 이상의 DBMS를 운영 가능한 사양

초기 구매비용 정도를 제외하면 저전력 컴퓨터인 raspberry는 유지비용이 거의 나오지 않는다. (한 달에 1000원 정도?) 때문에 위 조건을 만족하는 대부분의 클라우드 VM과 비교했을 때 라즈베리파이가 매력적으로 다가왔다. ~~이미 라즈베리파이가 있어서 초기 비용이 적었던 이유도 있었다.~~

| 사양      | 라즈베리파이     | AWS Lightsail |
| ------- | ---------- | ------------- |
| CPU     | 4Core      | 2Core         |
| RAM     | 8GB        | 8GB           |
| VGA     | 4K support | X             |
| Pricing | 100,000 정도 | 40$/per month |
### 2. 학습
서버를 다루고 여러 OS를 접한다는 것도 하나의 개발 경험이 되어 다른 분야의 개발에서도 도움이 될 것이라 생각한다. 실제로 육군에서 Unix 서버를 관리해본 경험이 있어 CLI나 네트워크 개념에 익숙해져 있었고, 수업이나 프로젝트에서 해당 개념들이 활용되는 것을 보며 느낀 바가 있어 직접 운영해보고 싶었다.

### 3. 커뮤니티

비교적 고사양의 미니PC들을 선택하지 않은 이유.
2번과도 이어지는 면이 있는데, 직접 서버를 구축하고 운영하는 과정에서 발생하는 에러들을 스스로 해결하기 위해서는 서칭이 필수적이기 때문이다. 라즈베리파이는 오랜 기간동안 다양한 개발자들에게 사용되었고, 그 중 발생한 문제와 비슷한 것들을 많이 해결한 사람들의 텍스트들이 흩뿌려져 있어 해결이 편한 것이 큰 장점이라 생각한다. AI를 사용해 개발하더라도 명확한 출처(포럼이나 StackOverflow 등)가 있는 편이 배우기에는 더 나았다 생각한 면도 있다.
