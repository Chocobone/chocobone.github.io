### 개인 블로그!

개발하면서 알게 되는 지식들이나 정보들을 요약해 저장해 놓으면 나중에 필요할 때 활용하기 편할 것 같다 생각해 블로그 사이트들을 찾아보았다...만, 네이버 블로그는 글 작성하기가 귀찮아지는 UI라 싫었고, velog나 tistory 중 고민하던 와중 우연하게 Obsidian의 CEO인 kepano의 [블로그 포스트를 보게 되었다](https://www.threads.com/@kepano/post/DDxmyWzOKp0?xmt=AQGzbVXXjv9Q0CIid-bRmSBo1EDEpnRtGOVVW1_YdLxkHw).

해당 블로그의 핵심은 Obsidian과 Jekyll을 활용한 포스팅 및 github을 사용한 배포. 별도의 사이트 접속 필요없이 git clone한 repo를 작성하고 push하면 내용이 업데이트되기에 pull/push를 제외하고는 local로 작성/저장이 가능한 장점이 있다. Obsidian도 원래 사용하고 있었고, Jekyll도 크게 어렵지 않은데다 Gemini Pro도 있어서 개발 블로그 틀을 직접 만드는 게 낫겠다 생각해 직접 블로그를 만들게 되었다.

개발 및 배포와 관련해서는 [해당 블로그](https://blog.itcode.dev)의 라즈베리파이 개시물을 주로 참고했으며, 블로그 작성 방식? 철학에서는 [kepano의 블로그](https://stephango.com)와 [escriboy 블로그](https://escriboy.com/%EC%98%B5%EC%8B%9C%EB%94%94%EC%96%B8%EC%9C%BC%EB%A1%9C-%EB%B8%94%EB%A1%9C%EA%B7%B8-%EC%9E%91%EC%84%B1%ED%95%98%EA%B8%B0)의 도움을 많이받았다.


###  목표사항 확인

블로그 개발을 마음먹었을 때 최종적인 목표는 다음과 같았다.

- 배포 서버(라즈베리파이) 세팅
- 도매인 구매 및 인증서 설정
- Jekyll 테마 설정 및 블로그 꾸미기 

라즈베리파이는 24년부터 가지고 있어 어떻게든 사용할 방법을 찾고 있었고, 블로그 구축에 필요한 배포나 도메인 서버로 활용하면 좋을 것 같아 사용했다. 특히 도메인 서버만 사용하는게 아니라 추가적인 백엔드 서버로도 사용 가능할 것 같아, 개발자 언저리(..) 입장에서도 마음에 들었다.
도메인도 보통 개발 블로그 도메인으로 사용하는 github.io는 식상할 것 같아 별도로 구매했다.

블로그를 만들어가면서 경험했던 내용들이 유용하고 개발하는데 좋은 경험라 생각해서 몇 없는 사진들과 같이 적어볼 예정이다.
