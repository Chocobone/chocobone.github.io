초기 설정을 마쳤지만, 안정적으로 서버를 운영하기 위해 보안 등 추가적인 설정들과 에러 해결이 필요했다. 
크게 편의성을 위한 부분과 보안을 위한 부분으로 나뉜다. 보안의 경우 아래 작업들을 진행해도 해커들이 진심으로 공격하면 쉽게 뚫리지만, 해당 보안 대책들로 뚫기 '귀찮게' 만들면 표적이 될 위험이 줄어들기에 필수적이다.

- 아래 설정을 진행하기전 업데이트를 진행하는 것이 좋다.

```bash
sudo apt update && sudo apt upgrade -y
sudo apt autoremove #불필요한 패키지 삭제
```

## fix 1: ip 고정

초기 설정을 진행한 다음날 ssh를 사용해 서버에 접근이 안되는 문제가 있었다. 공유기 게이트웨이까지 들어가 확인한 결과, 서버의 ip가 최초 설정과 달라 `ssh 사용자@[IP 주소]` 명령어로 접근이 힘들었던 것이었다. 원인을 찾아보니, 유/무선 관계없이 공유기 내의 DHCP 설정이 켜져 있는 경우 인터넷에 접속할 때마다 IP가 바뀌어서 그렇다 들어 IP를 고정하는 것이 좋아보였다.
[해당 블로그](https://passing-story.tistory.com/entry/Linux-%EC%9A%B0%EB%B6%84%ED%88%AC-%EA%B3%A0%EC%A0%95-IP-%EC%84%A4%EC%A0%95-Ubuntu-Static-IP-%EC%84%A4%EC%A0%95)를 참고했다. `ifconfig` 명령어를 사용해 현재 인터넷 연결상태를 주기적으로 확인하며 진행했다.

```bash
sudo vi /etc/netplan/00-installer-config.yaml
```

다음 형식으로 코드를 수정하니 재부팅해도 문제없이 접속 가능하다. 
```bash
network:
  ethernets:
    ens33:
      dhcp4: no
      addresses : [192.168.100.100/24] # IP 주소
      gateway4 : 192.168.100.1 #대부분 끝자리 1
      nameservers:
	      addresses : [8.8.8.8, 4.4.4.4] #공통
  version: 2
```

아니면 DHCP 설정을 변경해도 고정 가능하다. 라우터 설정에서 MAC 주소에 맞춰 IP를 할당하면 해당 기기로 접속했을 때 동일한 IP를 부여해준다. 다만 공유기 전원이 꺼지면 초기화되니 주의할 것.

## fix 2: GUI 활성화

ssh 접근이 가능하지만, 세부 설정이나 그래픽이 필요한 프로그램(Tomcat)을 사용하는 경우를 위해 원격 접속이 필요하다 느꼈다. xrdp 패키지를 사용하면 윈도우의 "원격 데스크톱 관리"를 통해 편하게 접속이 가능하다. 참고로 GUI를 사용하는 경우 Ubuntu Desktop 버젼을 설치해야 한다. **Ubuntu Server의 경우 그래픽 자체가 없으므로** 설치할 때 주의하자

```bash
sudo apt-get install xrdp

sudo vi /etc/polkit-1/localauthority/50-local.d/45-allow-colord.pkla
```

```pkla
#45-allow-colord.pkla
[Allow Colord all Users]

Identity=unix-user:*

Action=org.freedesktop.color-manager.create-device;org.freedesktop.color-manager.create-profile;org.freedesktop.color-manager.delete-device;org.freedesktop.color-manager.delete-profile;org.freedesktop.color-manager.modify-device;org.freedesktop.color-manager.modify-profile

ResultAny=no

ResultInactive=no

ResultActive=yes
```

```bash
service xrdp start #재시작해서 변경사항을 적용한다
```

## fix 3: SSH Port 변경
SSH를 처음 설치했을 때 기본 포트는 22/tcp를 사용한다. 보안 측면에서 22를 그대로 사용하면 비밀번호만 뚫려도 다른 사용자가 내 컴퓨터에 접근 가능하기 때문에 별도의 포트를 사용하면 보안이 "손쉽게" 뚫리지는 않게된다.
```bash
sudo vi /etc/ssh/sshd_config
```
```bash
# /etc/ssh/sshd_config

#문서 중간에 위치한 #Port 22를 주석해제하고 원하는 포트를 사용하면 된다
# Port 22 <- 주석해제
Port {포트번호}
```

## fix 4: 방화벽 설정(ufw)

드디어 등장한 네트워크 뉴비 통곡의 벽(개인적). 원격 접속은 SSH 포트나 xrdp 포트를 사용해야 연결되기 때문에, 포트를 잘못 막아버렸다가 원격 접속이 안되는 경우가 발생할 수 있다. 이 경우 서버에 직접 연결해서 수정하는 귀찮음 X 100 이벤트가 발생하기에 정신 차리고 진행해야 한다.

```bash
sudo apt install ufw
sudo systemctl status ufw
```

기본 정책의 경우 외부 요청을 내부(서버)에서 받아들이는 것이 `incoming`, 내부에서 외부로 요청을 보내는 것이 `outgoing`이다. allow된 port를 통해서만 해당 서버로 요청을 보내는것이 가능하도록 설정했다. SSH 포트의 경우 위에서 변경한 포트로 진행했다. 
```bash
# 1. 기본 정책 설정 (모든 incoming 차단, outgoing 허용) 
sudo ufw default deny incoming 
sudo ufw default allow outgoing 

# 2. SSH (22), HTTP (80), HTTPS (443) 허용
sudo ufw allow {ssh 포트번호}/tcp 
sudo ufw allow 80/tcp 
sudo ufw allow 443/tcp

# 3. UFW 활성화 
sudo ufw enable 

# 4. 설정 확인
sudo ufw status verbose

# 5. 재시작을 통한 설정 적용
sudo systemctl restart ufw
```
